# P07 — Event Prep

## Purpose

Prepare a structured note before a scheduled company or macro event. The output should clarify what is known, what expectations appear to be, and what data points should be reviewed after the event.

## Required inputs

- Event name.
- Date and time.
- Relevant names or themes.
- User-provided context or prior notes.

## Source checklist

1. Official event calendar or company investor relations page.
2. Prior event release and transcript, if relevant.
3. Consensus/context data from dashboards if available.
4. News and commentary only for narrative framing.

## Output format

```markdown
# Event Prep — <Event>

## 1. Event Summary

## 2. What Matters
| Metric/topic | Why it matters | Prior value/context | Source |
|---|---|---|---|

## 3. Expectations Map

## 4. Scenario Map
| Scenario | What would drive it | What to review after event |
|---|---|---|

## 5. Names or Themes to Watch

## 6. Post-Event Checklist

## 7. Sources Used

## 8. Missing Data
```

## Quality checks

- Distinguish scheduled facts from expectations.
- Use official calendars for event timing.
- Avoid pretending to know the event outcome in advance.
