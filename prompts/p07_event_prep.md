# P07 — Event and Earnings Prep

## Purpose

Prepare for a scheduled event such as earnings, CPI, jobs report, FOMC, investor day, product launch, or regulatory decision.

## Required inputs

- Event name.
- Date/time.
- Affected names or themes.
- User context.
- Prior notes if available.

## Source checklist

1. Official calendar or company investor relations page.
2. Prior event release/transcript.
3. Consensus expectations if available.
4. Historical reaction if available.
5. Implied move or volatility context if user provides it or a source is available.
6. News/commentary for narrative context only.

## Output format

```markdown
# Event Prep — <Event>

## 1. Event Summary

## 2. What Matters Most
| Metric/topic | Why it matters | Prior value/context | Source |
|---|---|---|---|

## 3. Expectations Map
What does the market appear to care about? Distinguish evidence from assumption.

## 4. Positioning / Volatility Context
Include implied move, volatility, or positioning only if sourced or provided.

## 5. Scenario Map
| Scenario | What drives it | Likely interpretation | What to review after |
|---|---|---|---|

## 6. Affected Names / Themes

## 7. Post-Event Checklist

## 8. Sources Used

## 9. Missing Data
```

## Quality checks

- Distinguish scheduled facts from expectations.
- Use official calendars for event timing.
- Avoid pretending to know the event outcome in advance.
- Tie the event back to the user's supplied watchlist or portfolio context when possible.
