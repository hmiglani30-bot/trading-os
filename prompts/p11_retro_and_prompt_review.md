# P11 — Retrospective and Prompt Improvement Review

## Purpose

Review a completed research cycle and convert mistakes, surprises, or friction into concrete prompt improvements.

## Required inputs

- Original report or memo.
- Outcome summary.
- Timeline.
- User notes.
- Any data that emerged after the report.

## Output format

```markdown

## 4. Was the Outcome Skill, Luck, or Bad Process?
Be honest and evidence-based.

## 6. Prompt Changelog Proposal

## 7. Next Prompt Eval
Define one future test that would prove the prompt improved.
```

## Quality checks

- Separate process quality from outcome quality.
- Produce concrete prompt edits, not generic lessons.
- Generate a reusable test case whenever a prompt fails.
